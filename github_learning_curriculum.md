# GitHub学習カリキュラム

## 🎯 学習ゴール
GitHubの基本から応用まで、実践を通じてマスターする

## 📚 カリキュラム構成

### 【Phase 1】GitHub Web インターフェース入門（1-2日）
**目標：** ブラウザ上でGitHubの基本操作ができるようになる

#### Lesson 1.1: 最初のリポジトリ作成
- リポジトリとは何かを理解する
- 新しいリポジトリを作成する
- READMEファイルを作成・編集する
- **実習：** 自己紹介リポジトリを作る

#### Lesson 1.2: ファイル操作の基本
- ファイルの追加・編集・削除
- コミットメッセージの書き方
- ファイル履歴の確認
- **実習：** テキストファイルを追加して編集履歴を作る

#### Lesson 1.3: GitHub Issuesの使い方
- Issueとは何か
- Issueの作成と管理
- ラベルとマイルストーン
- **実習：** 学習タスク管理用のIssueを作る

---

### 【Phase 2】Git基礎とローカル環境（3-4日）
**目標：** 自分のパソコンでGitを使えるようになる

#### Lesson 2.1: Git環境のセットアップ
- Gitのインストール（Windows/Mac）
- 初期設定（ユーザー名、メールアドレス）
- SSH鍵の設定
- **実習：** Gitをインストールして設定を完了する

#### Lesson 2.2: クローンとプッシュの基本
- `git clone`: リポジトリをダウンロード
- `git add`: 変更をステージング
- `git commit`: 変更を記録
- `git push`: GitHubへアップロード
- **実習：** Phase 1で作ったリポジトリをクローンして編集

#### Lesson 2.3: プルと同期
- `git pull`: 最新の変更を取得
- `git status`: 現在の状態確認
- `git log`: 履歴の確認
- **実習：** Web版とローカル版の両方で編集して同期する

#### Lesson 2.4: .gitignoreの使い方
- .gitignoreとは
- 無視すべきファイルの種類
- .gitignoreの書き方
- **実習：** 適切な.gitignoreファイルを作成する

---

### 【Phase 3】ブランチ戦略（3-5日）
**目標：** ブランチを使った開発フローを理解する

#### Lesson 3.1: ブランチの基本概念
- ブランチとは何か、なぜ必要か
- mainブランチの役割
- `git branch`: ブランチ一覧確認
- `git checkout`: ブランチ切り替え
- **実習：** 新しいブランチを作成して切り替える

#### Lesson 3.2: ブランチでの作業
- `git checkout -b`: ブランチ作成と切り替えを同時に
- ブランチでの編集とコミット
- ブランチのプッシュ
- **実習：** featureブランチで新機能を作る

#### Lesson 3.3: マージの基本
- `git merge`: ブランチの統合
- Fast-forwardマージ
- マージコミットの作成
- **実習：** featureブランチをmainにマージする

#### Lesson 3.4: コンフリクトの解決
- コンフリクトとは何か
- コンフリクトが起こる状況
- コンフリクトの解決方法
- **実習：** 意図的にコンフリクトを起こして解決する

---

### 【Phase 4】Pull Requestとコラボレーション（4-6日）
**目標：** チーム開発の基本的な流れを習得する

#### Lesson 4.1: Pull Requestの作成
- Pull Request（PR）とは
- PRの作成手順
- 良いPR説明の書き方
- **実習：** 自分のリポジトリでPRを作る

#### Lesson 4.2: コードレビューの基礎
- レビューの重要性
- コメントの書き方
- 変更のリクエスト
- **実習：** 自分のPRにコメントを付けてみる

#### Lesson 4.3: Forkとコントリビューション
- Forkの概念
- 他人のリポジトリへの貢献方法
- アップストリームの設定
- **実習：** オープンソースプロジェクトをForkする

#### Lesson 4.4: GitHub Actionsの基礎
- CI/CDとは
- GitHub Actionsの基本
- 簡単なワークフローの作成
- **実習：** 自動テストのワークフローを設定する

---

### 【Phase 5】応用とベストプラクティス（3-5日）
**目標：** 実務で使える技術を身につける

#### Lesson 5.1: Git応用コマンド
- `git rebase`: 履歴の整理
- `git stash`: 一時的な変更の保存
- `git cherry-pick`: 特定のコミットを適用
- **実習：** 各コマンドを実際に使ってみる

#### Lesson 5.2: リリース管理
- タグの作成と管理
- セマンティックバージョニング
- リリースノートの作成
- **実習：** v1.0.0リリースを作成する

#### Lesson 5.3: プロジェクト管理機能
- GitHub Projects
- マイルストーン管理
- チーム協力のワークフロー
- **実習：** プロジェクトボードを作成する

#### Lesson 5.4: トラブルシューティング
- よくある問題と解決方法
- `git reset`と`git revert`の使い分け
- 間違ったコミットの修正
- **実習：** 様々な問題を解決する

---

### 【Phase 6】実践プロジェクト（1-2週間）
**目標：** 学んだことを統合して実際のプロジェクトを完成させる

#### 最終課題
以下のいずれかのプロジェクトを選んで完成させる：
1. 個人ブログサイト（HTMLとCSSで作成）
2. ToDoアプリ（お好みの言語で）
3. ドキュメントサイト（Markdownで作成）

**要件：**
- 適切なブランチ戦略の使用
- Pull Requestでの開発
- Issueでのタスク管理
- READMEとドキュメントの整備
- 適切なコミットメッセージ
- GitHub Actionsの活用（任意）

---

## 📝 学習の進め方

1. **各Lessonの前に：** その日の学習目標を確認
2. **実習中：** 手を動かして実際に操作する
3. **各Lessonの後に：** 学んだことを自分の言葉でメモ
4. **Phase終了時：** 振り返りと理解度チェック

## 🔧 必要なツール

- GitHubアカウント（取得済み✓）
- テキストエディタ（VS Code推奨）
- ターミナル/コマンドプロンプト
- Git（Phase 2でインストール）

## 📌 学習のコツ

- 焦らず一つずつ確実に理解する
- 分からないことは遠慮なく質問する
- エラーを恐れない（エラーから学ぶことが多い）
- 毎日少しずつでも触り続ける
- 実際に手を動かすことが最重要

- 最終課題プロジェクトを完成させる

- Pull Requestベースの開発フローを理解している
